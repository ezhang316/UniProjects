# 3900: CS Capstone Project – Soup Kitchen Application

Full-stack web application developed as a team capstone project. A platform for managing soup kitchen donations, volunteer coordination, and meal distribution.

## Architecture

### Frontend (TypeScript & React)
**Technology Stack:**
- React 18 with TypeScript for type-safe component development
- React Router for navigation and multi-page functionality
- Bootstrap & React-Bootstrap for responsive UI
- Draft.js for rich text editing capabilities
- Axios for API communication

**Key Features:**
- Responsive component architecture
- Rich text editing for meal descriptions
- Form handling and validation
- Authentication state management
- Dynamic data fetching and display

### Backend (Spring Boot & Java)
**Technology Stack:**
- Spring Boot 2.7.4 with Spring MVC
- Spring Data JPA for ORM and database interaction
- MySQL database integration
- RESTful API design
- Jackson JSON processing

**Key Components:**
- Controllers for request handling and routing
- Repositories for database abstraction
- Service layer for business logic
- Database models for donations, volunteers, and meals
- Authentication and authorization

### Database (MySQL)
- Relational schema for meals, donations, volunteers
- Foreign key relationships and data integrity constraints
- Query optimization for reporting features

## Technical Achievements

- **Full-stack integration:** Seamless communication between React frontend and Spring Boot backend
- **Type safety:** TypeScript enforces compile-time type checking
- **Scalable architecture:** Separation of concerns with service and repository layers
- **Responsive design:** Mobile-friendly interface using Bootstrap
- **Data persistence:** Robust database design with proper normalization

## Development Areas

- Team collaboration and version control (Git)
- API design and RESTful principles
- Component state management in React
- Database schema design and migrations
- Authentication and session management
